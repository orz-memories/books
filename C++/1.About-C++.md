## 1.3
众所周知, 计算机只能理解二进制的指令. 二进制的指令就是CPU可直接执行的指令的集合, 也叫机器码[3].  
机器码是计算机CPU直接可执行的命令, 和CPU是密切相关的, 甚至每种CPU可能有不同的机器码.

    .file   "main.cpp"
    .lcomm __ZStL8__ioinit,1,1

> 汇编语言示例, 节选自同样的小程序

从上面汇编语言的例子中可以看出, 结构较机器码清晰了许多, 可以直接书写十进制数字, 可以直接书写英文词句(比如Hello World)[5], 用一些字母缩写代替了二进制的指令.

应该说汇编语言和机器码存在一一对应的关系, 它们之间可以轻松地进行转换.

但是CPU依旧只能理解机器码, 所以我们需要把汇编语言转换成机器码, 这个工作通常由程序来做[6].  
这种程序叫`汇编器`, 可以把汇编语言转换成机器码. 还有`反汇编器`, 用来把机器码转换成汇编语言.

使用汇编语言让程序员省去了机器码编程中的一些繁琐步骤, 但依然没有本质性的进步.

    #include <stdio.h>
    int main()
    {
        printf("Hello World!\n");
        return 0;
    }

> C语言代码示例, 同样的小程序.

丹尼斯·里奇[7]在1970年发明了如上的C语言[8], 应该说和汇编语言相比, C语言已经非常接近自然语言了[9].

同样, 计算机不认识C语言, 只认识机器码, 所以也需要一个软件将C语言翻译为机器码.  
这个翻译的过程叫`编译`, 用于编译的程序叫做`编译器`, 它可以把C语言代码编译成机器码.

十几年以后, Bjarne Stroustrup又改进了C语言, 改进后的语言被称为C++.  

    #include <iostream>
    using namespace std;

    int main()
    {
        cout<<"Hello World!"<<endl;
        return 0;
    }

## 1.4
C++并不属于某个组织或个人, 任何人都可以去使用它, 时间长了就容易出现混乱.

1990年开始ISO[12]和ANSI[13]就开始着手将C++的内容进行汇总, 制定具有知道意义的"标准"文章, 由于C++的内容实在太多太复杂, 个别部分有很大争议, 这项工作历时8年才完成. 1998年, ISO和ANSI发布了C++的第一个标准, C++98.

随后在2003年, C++的第二个标准C++03被发布, C++03没有新增内容, 只是对C++98中有争议的部分做了修订, 并修正了一些漏洞. 到目前位置, C++03是被广泛应用的版本, 我们将以它为主来学习C++.

C++11是近期才发布的新标准, 新增了很多的功能, 但是编译器厂商还在忙着完善编译器, 程序员还在学习, 目前应用不是很广泛, 我们将会略微提及有关C++11的内容.

> [12] ISO : International Organization for Standardization, 国际标准化组织.

> [13] ANSI : American National Standards Institute, 美国国家标准学会.

## 1.5
C++的应用领域非常广泛, 你所见到的大部分桌面软件都是C++编写的(如QQ, 迅雷等, Chrome), 还有一些服务器软件(如MySQL, Mongo), 以及大部分的游戏引擎, 都是C++编写的.  
相比之下C++既有C的运行效率, 又有高级语言的结构组织能力, 不过这些市场也在慢慢地被Go, Python等开发效率更高的高级语言夺走, 因为现在计算机都白菜价了.

虽然C++有这么缤纷的应用, 但是基础是一定要打好的, 所以恐怕在今后的很长时间, 我们都需要和所谓的"黑框框"[14]打交道.  
不要小瞧它哦, 在之后的几节课中, 我们会利用它完成复杂的数学计算, 还可以编写有趣的文字游戏.
